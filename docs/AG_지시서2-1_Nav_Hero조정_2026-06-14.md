## STEP 1. Nav 레이아웃 — 로고 좌측 이동 (Nav.astro)

### 현재 문제
`.nav-menu`에 `order: -1; flex: 1`이 걸려 메뉴가 맨 왼쪽을 차지하고 로고가 우측으로 밀림.

### 수정: 데스크톱 배치 순서를 [로고 | 메뉴(중앙) | 스위처(우)]로
```css
.nav-inner {
  display: flex;
  align-items: center;
  gap: 1.25rem;
}
.nav-brand   { order: 0; flex-shrink: 0; margin-right: auto; } /* 좌측 고정 */
.nav-menu    { order: 1; flex: 0 1 auto; }   /* order:-1, flex:1 제거 */
.lang-switch { order: 2; margin-left: auto; flex-shrink: 0; } /* 우측 고정 */
```
- `.nav-menu`의 기존 `order: -1; flex: 1;` **삭제**.
- 로고 `margin-right: auto` 또는 스위처 `margin-left: auto`로 메뉴를 중앙~우측에 자연 배치.
- 최종 시각: **GSFGrace(좌) ……… 메뉴5개 ……… 日本語|한국어(우)**

> ※ 모바일(<768px) 순서는 현행 유지: 로고(좌) — 스위처 — 햄버거(우). 정상 작동 중이므로 건드리지 말 것.

## STEP 2. 메뉴 highlight 박스 제거 (Nav.astro)

`はじめての方へ`의 olive 채움 박스 제거, 전 메뉴 동일 톤:
```css
/* .nav-link.highlight 규칙 삭제 또는 아래로 약화 */
.nav-link.highlight {
  background: transparent;
  color: var(--color-accent);   /* 텍스트만 olive로 살짝 구분 */
  font-weight: 600;
}
.nav-link.highlight:hover {
  background: var(--color-accent-soft);
  color: var(--color-accent);
}
```
- 또는 navItems에서 `highlight: true` 자체를 제거하고 일반 링크로 통일해도 무방.
- 이유: 화면 중앙에 큰 CTA 버튼(`はじめての方へ →`)이 이미 있어 중복 강조 불필요.

## STEP 3. 로고 명시성 강화 (Nav.astro)

박스 없이 **굵게 + 크게**로:
```css
.brand-name {
  font-size: 1.25rem;        /* 1rem → 1.25rem */
  font-weight: 800;          /* 700 → 800 */
  color: var(--color-accent);
  letter-spacing: -0.01em;
  font-family: var(--font-serif);
}
.brand-sub {
  font-size: 0.68rem;        /* 약간만 키움 */
  color: var(--color-muted);
  letter-spacing: 0.08em;
}
```
> 이 단계는 박스 없이 굵게/크게만. 적용 후에도 약하면 그때 박스 검토(추후 판단).

## STEP 4. Hero 전체 배경 수채화 (index.astro + ko/index.astro)

### 4-1. 이미지 배치
- ChatGPT 수채화 PNG를 `public/images/hero-nihonbashi.png` (또는 .webp 권장)로 저장.
- Hero 섹션 배경으로 적용:
```css
.hero {
  position: relative;
  padding: 5rem 1.5rem 4rem;
  text-align: center;
  background-image: url('/images/hero-nihonbashi.png');
  background-size: cover;
  background-position: center right;  /* 시안처럼 그림 오른쪽 강조 */
  border-radius: 0;
}
/* 가독성 오버레이 — 텍스트 영역 보호 (필수) */
.hero::before {
  content: '';
  position: absolute;
  inset: 0;
  background: linear-gradient(
    to right,
    rgba(250, 248, 243, 0.88) 0%,
    rgba(250, 248, 243, 0.70) 45%,
    rgba(250, 248, 243, 0.30) 100%
  );
  z-index: 0;
}
.hero > * { position: relative; z-index: 1; }
```
- **오버레이는 필수.** 명조 검은 텍스트가 수채화 위에서 읽혀야 함. 위 그라데이션은 왼쪽(텍스트) 진하게, 오른쪽(그림) 투명하게.
- 모바일에서는 그림이 약하게: `background-position: center; ` + 오버레이 전체 `0.82` 균일하게 (미디어쿼리로).

### 4-2. 가독성 검증
- Hero 텍스트(GOOD SAMARITAN FLOURISHING / Receive Life. Walk Together. / 성구 / 버튼)가 **모든 화면폭에서 명확히 읽히는지** 확인.
- 안 읽히면 오버레이 불투명도를 높일 것.

### 4-3. About 섹션 placeholder
- About 좌측의 `nihonbashi-placeholder.svg`는 **그대로 유지** (Hero에 수채화가 가므로 About은 추후 실사진용 자리로 보존).
- 또는 About 이미지를 제거하고 텍스트 1단으로 바꿀지는 추후 결정 — 이번엔 건드리지 말 것.
