## 0. 배경 — 왜 이 작업이 먼저인가

콘텐츠 편집(ChatGPT)·배포 파이프라인을 가동하기 전에, **콘텐츠 레이어의 라우팅이
미완성**임을 코드 확인으로 발견했다. 이 상태로 글을 올리면 페이지가 안 뜨거나
JA/KO가 섞인다. 따라서 콘텐츠 투입 전 아래 2개 과제를 먼저 처리한다.

### Claude가 확인한 현재 실태 (사실)

1. **상세 라우터 부재**: `devotion`·`walk`·`arche`·`faith-life`는 `index.astro`(목록)만
   있고 **개별 글 상세 라우터(`[slug].astro`)가 없다.** index는 `/devotion/{slug}`로
   링크하지만 그 경로를 받는 파일이 없어 **404**가 된다. (`qa`만 `[tag]/[slug].astro` 존재)

2. **lang 분리 미구현**: 모든 index가 `getCollection(cat, ({data}) => !data.draft)`로
   **lang 필터 없이** 전체 글을 가져온다. 그 결과 JA 글이 `/ko/...`에도, KO 글이
   `/...`에도 노출된다. JA/KO가 분리되지 않는다.
   - 예: `ko/qa/index.astro`와 `qa/index.astro`가 동일한 글 집합을 표시.

3. **API 확인 필요**: 기존 코드가 `post.slug`를 쓰는데, content.config.ts는
   `glob` loader 기반이다. 최신 Astro에서는 `post.id`가 표준. 빌드 시
   `post.slug`가 deprecated/undefined인지 확인하고 필요 시 `post.id`로 통일.

## TASK-0A. lang 분리 정책 구현 (모든 collection index)

각 카테고리 index에서 **현재 언어에 맞는 글만** 필터링한다.

- JA 페이지(`src/pages/{cat}/index.astro`): `({data}) => !data.draft && data.lang === 'ja'`
- KO 페이지(`src/pages/ko/{cat}/index.astro`): `({data}) => !data.draft && data.lang === 'ko'`
- 대상: devotion, walk, arche, qa, (faith-life는 보류지만 동일 패턴으로 맞춰둠)
- qa의 KO/JA index 둘 다 동일하게 lang 필터 추가.

## TASK-0B. 개별 글 상세 라우터 신설

`qa/[tag]/[slug].astro`를 **참조 패턴**으로 아래 라우터를 만든다.
(qa 라우터의 레이아웃·CSS·story CTA 구조를 재사용. CTA 문구만 카테고리·언어에 맞게.)

| 신설 파일 | lang 필터 | URL |
|-----------|-----------|-----|
| `src/pages/devotion/[slug].astro` | ja | `/devotion/{slug}` |
| `src/pages/ko/devotion/[slug].astro` | ko | `/ko/devotion/{slug}` |
| `src/pages/walk/[slug].astro` | ja | `/walk/{slug}` |
| `src/pages/ko/walk/[slug].astro` | ko | `/ko/walk/{slug}` |
| `src/pages/arche/[slug].astro` | ja | `/arche/{slug}` |
| `src/pages/ko/arche/[slug].astro` | ko | `/ko/arche/{slug}` |

- `getStaticPaths`에서 `getCollection(cat, ({data}) => !data.draft && data.lang === 'ja'|'ko')`로
  언어별 경로만 생성.
- qa의 기존 `[tag]/[slug].astro`도 lang 필터 적용 + **KO용 라우터(`src/pages/ko/qa/[tag]/[slug].astro`) 신설**
  (현재 KO qa index는 `/ko/qa/{tag}/{slug}`로 링크하나 해당 KO 상세 라우터가 없을 가능성 → 확인 후 신설).

## TASK-0C. API 정합성

- `post.slug` 사용처를 점검. glob loader에서 `post.id`가 표준이면 전부 `post.id`로 통일.
- `Content` 렌더링: `const { Content } = await render(post)` (Astro 최신) 또는
  기존 `post.render()` 중 빌드 통과하는 방식으로 통일.
