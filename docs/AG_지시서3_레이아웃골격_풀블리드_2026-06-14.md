# AG 작업 지시서 #3 — GSFGrace 레이아웃 골격 정비 (풀블리드 Hero + 폭 통일)
> Created: 2026-06-14
> 담당: AG (AntiGravity)
> 검증: Claude → 승주 목사님
> 전제: 지시서 #2-2까지 적용 완료(라이브 확인). 본 지시서는 "밸런스 결함" 구조 수정.
> 참조 디자인: gsfark.com (넓지만 안 휑한 폭 밸런스)

---

## 진단 (왜 "애매한" 느낌이었나)

1. **폭 기준 3개 혼재**: Nav 1100px / Hero 720px 박스 / About 640px → 정렬선 어긋남
2. **Hero 이미지가 720px 박스에 갇혀 둥둥 뜸** → 미니멀도 풀임팩트도 아닌 중간 상태
3. **언어 스위처가 1100px 끝으로 동떨어짐** → Nav만 넓어서

## 해결 원칙

- **레이아웃 폭 = 1080px 단일 기준** (Nav·섹션 컨테이너 공통)
- **Hero만 100vw 풀블리드** (수채화 화면 끝까지)
- **순수 텍스트 칼럼 = 720px** (가독성)
- 정렬선을 1080px 하나로 통일 → 밸런스 확보

---

## STEP 1. 전역 폭 변수 + 색 강화 (BaseLayout.astro)

### 1-1. 폭 변수 추가 (:root)
```css
:root {
  /* ... 기존 토큰 유지 ... */
  --width-container: 1080px;   /* 레이아웃 폭 (Nav·섹션) */
  --width-text: 720px;         /* 순수 텍스트 칼럼 */
}
```

### 1-2. 색 강화 (또렷하게 — 사용자 요청)
```css
--color-text:  #1C1A17;   /* #292524 → 더 진하게 */
--color-muted: #57534E;   /* #78716C → 확실히 진하게 (부제·성구 또렷) */
```

### 1-3. main 폭 조정
```css
main {
  min-height: 70vh;
  max-width: var(--width-container);   /* 720 → 1080 */
  margin: 0 auto;
  padding: 0 1.5rem;
}
```
> 단, main 안의 텍스트 전용 섹션(About 등)은 STEP 4에서 720px로 별도 제한.

---

## STEP 2. Nav 폭 통일 — 스위처 동떨어짐 해결 (Nav.astro)

### 2-1. nav-inner 폭을 1080px로
```css
.nav-inner {
  max-width: var(--width-container);   /* 1100 → 1080, 변수화 */
  margin: 0 auto;
  padding: 0 1.5rem;
  display: flex;
  align-items: center;
  gap: 1.25rem;
  height: 64px;
}
```
- 이제 Nav가 본문과 같은 1080px 정렬선 → 스위처가 콘텐츠 우측 끝에 자연히 붙음.

### 2-2. 로고와 메뉴 간격 + 메뉴 자체 간격
```css
.nav-menu {
  margin-left: 2rem;   /* 로고와 메뉴 사이 숨통 */
  gap: 0.5rem;         /* 0.2rem → 0.5rem, 메뉴 항목 간격 */
}
```

## STEP 3. 로고 박스 처리 (Nav.astro) — 사용자 요청

"굵게"로는 명시성 부족 → **박스 확정**.
```css
.nav-brand {
  display: inline-flex;
  flex-direction: column;
  justify-content: center;
  line-height: 1.15;
  text-decoration: none;
  flex-shrink: 0;
  background: var(--color-accent);   /* olive 박스 */
  padding: 0.45rem 0.85rem;
  border-radius: 4px;
}
.brand-name {
  font-size: 1.15rem;
  font-weight: 800;
  color: #fff;                       /* 박스 안 흰색 */
  letter-spacing: -0.01em;
  font-family: var(--font-serif);
}
.brand-sub {
  font-size: 0.62rem;
  color: rgba(255,255,255,0.82);     /* 흰색 약하게 */
  letter-spacing: 0.1em;
  font-family: var(--font-sans);
}
```
- radius 4px로 사이트 톤 통일.
- 모바일에서도 동일 박스 유지.

## STEP 4. Hero 풀블리드 (index.astro + ko/index.astro)

### 4-1. Hero를 화면 끝까지 (100vw)
main 안에 있어도 풀블리드가 되도록:
```css
.hero {
  position: relative;
  width: 100vw;
  left: 50%;
  right: 50%;
  margin-left: -50vw;
  margin-right: -50vw;
  padding: 6rem 1.5rem 5rem;
  text-align: center;
  background-image: url('/images/hero-nihonbashi.webp');
  background-size: cover;
  background-position: center;
  overflow: hidden;
}
```
> 이 기법(`width:100vw; margin-left:-50vw; left:50%`)으로 부모 폭(1080px)을 뚫고 화면 끝까지 확장.

### 4-2. Hero 내부 텍스트는 중앙 정렬, 1080px 제한
```css
.hero > * {
  position: relative;
  z-index: 1;
  max-width: var(--width-container);
  margin-left: auto;
  margin-right: auto;
}
```

### 4-3. 오버레이 — 풀블리드에 맞게 균일하게
좌우 끝까지 이미지가 보이므로, 텍스트 가독성을 위해 **중앙 집중형** 오버레이:
```css
.hero::before {
  content: '';
  position: absolute;
  inset: 0;
  background: radial-gradient(
    ellipse 60% 80% at center,
    rgba(250,248,243,0.92) 0%,
    rgba(250,248,243,0.80) 55%,
    rgba(250,248,243,0.55) 100%
  );
  z-index: 0;
}
```
- 중앙(텍스트) 진하게, 가장자리(수채화) 살림.
- 텍스트 가독성 우선. 안 읽히면 중앙값 0.92 → 0.95.

### 4-4. 모바일
```css
@media (max-width: 768px) {
  .hero { padding: 4rem 1.25rem 3.5rem; }
  .hero::before {
    background: rgba(250,248,243,0.88);   /* 균일 */
  }
}
```

## STEP 5. 본문 텍스트 칼럼 720px (index.astro + ko/index.astro)

Hero 아래 모든 콘텐츠(About, 링크카드, PathLine)는 텍스트 가독 폭으로:
```css
.section-about,
.section-links {
  max-width: var(--width-text);   /* 720px */
  margin: 0 auto;
}
```
- PathLine도 720px 기준(이미 그럴 것 — 확인).
- About은 지시서 #2-2에서 1단 텍스트로 변경됨 → 720px 중앙.
